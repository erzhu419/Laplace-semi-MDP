## Final Paper Experiment Status

1. XL scheduler suite `paper_xl_20260706_0659` finished on node001-node006 and is published into tracked output folders.
2. Aggregated coverage: large-scale compression 135 rows, random maze 360 rows, option frontier 648 rows, amortized multitask 192 rows, edge reward 384 rows.
3. The corridor-1024 horizon artifact was fixed by using an effectively unbounded local horizon; the published large-scale table now has zero error rows.
4. Adaptive top-k diagnostics now support the paper hierarchy: exact/frozen RD Green is the reference objective, adaptive feasible top-k is the main feasible-discovery backend, fixed top-k is the ablation envelope, and score optimality requires interval dominance.
5. General finite-MDP tests separate interface portability from the main finite-grid compression claim. FrozenLake is favorable, PointMaze and MiniGrid reveal option-family bias, and Taxi-v4 is an explicit factored-variable failure with zero group-feasibility rate.
6. The P0 reviewer audit adds 405 paired planner rows, 1,560 direct-abstraction rows, 2,520 exhaustive-solver rows, 315 external-environment rows, and 162 end-to-end decomposition rows. The paper-facing tables use normalized gaps and label legacy timing denominators.
7. The boundary-proposal audit adds 360 graph-only reference contexts and 90 strict scale holdouts. A validation-selected single GCN with automatically selected CPU inference is `769.8x` faster than iterative selection and satisfies group feasibility plus normalized gap `<=0.01` on `68/90` rows, versus `62/90` for nearest-start and `71/90` for the adaptive RD reference proposal. Full audit still makes the learned path slower, so it remains an uncertified ablation.
8. The one permitted constraint-aware follow-up audits 1,159 unique fixed proposals. Train-only multi-head risk fitting raises strict scale-holdout raw joint passes to `81/90` at `656.0x` selection speed, against a candidate-family oracle of `85/90`. The paired table has 64 both-pass, 17 reranker-only, 7 reference-only, and 2 both-fail contexts (`p=0.0639`, exact McNemar); this is descriptive because objectives and guarantees differ. The validation-calibrated route catches only `3/9` test failures, and full audit catches all failures at `0.428x`; the predefined secondary-method gate therefore fails and the constraint-aware topology expansion is not run.
